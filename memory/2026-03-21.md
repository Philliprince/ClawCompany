1. **OpenClaw 工具使用方式**
   - sessions_spawn, sessions_history, sessions_send 是全局函数
   - 不需要导入，直接调用即可
   - 需要添加类型声明以获得 IDE 支持

2. **Jest + TypeScript 配置**
   - 使用 ts-jest preset
   - 需要正确配置 roots 和 testMatch
   - 全局变量 mock 方式：`global.sessions_spawn = jest.fn()`

3. **测试 Mock 策略**
   - Mock 返回的数据结构需要与真实 API 一致
   - 使用 mockResolvedValueOnce 链式 mock 多次调用
   - 默认值回退机制保证健壮性
